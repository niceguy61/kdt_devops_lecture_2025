# 🎯 팀별 다이어그램 챌린지 (60분)

## 📋 과제 개요

- **목표**: 각 팀이 실제 프로젝트 시나리오를 바탕으로 6가지 핵심 다이어그램을 작성
- **시간**: 작업 50분 + 발표 10분
- **도구**: Draw.io, Mermaid, MCP Python Diagram 중 자유 선택
- **결과물**: 팀당 6개 다이어그램 필수 제출

## 📊 필수 제출 다이어그램 (팀당 6개)

### 1. CI/CD 파이프라인 다이어그램
**요구사항**:
- 소스 코드부터 프로덕션 배포까지 전체 플로우
- GitHub Actions, Jenkins, 또는 AWS CodePipeline 활용
- 테스트, 보안 스캔, 승인 프로세스 포함
- 롤백 전략 표시

**평가 기준**:
- [ ] 완전한 CI/CD 플로우 표현
- [ ] 테스트 단계 포함
- [ ] 보안 검증 단계
- [ ] 배포 전략 명시

### 2. AWS 전체 아키텍처 다이어그램
**요구사항**:
- 전체 시스템의 AWS 서비스 구성
- 컴퓨팅, 스토리지, 데이터베이스, 네트워킹 서비스
- 고가용성 및 확장성 고려
- 비용 최적화 전략 반영

**평가 기준**:
- [ ] 적절한 AWS 서비스 선택
- [ ] 고가용성 설계
- [ ] 확장성 고려
- [ ] 보안 그룹 및 IAM 고려

### 3. VPC 네트워크 구조 다이어그램
**요구사항**:
- VPC, 서브넷, 라우팅 테이블 구성
- 퍼블릭/프라이빗 서브넷 분리
- NAT Gateway, Internet Gateway 배치
- 보안 그룹 및 NACL 설정

**평가 기준**:
- [ ] 논리적인 네트워크 분리
- [ ] 적절한 CIDR 블록 설계
- [ ] 보안 고려사항 반영
- [ ] 멀티 AZ 구성

### 4. MSA (마이크로서비스) 구조 다이어그램
**요구사항**:
- 비즈니스 도메인별 서비스 분리
- API Gateway 및 서비스 메시 구성
- 데이터베이스 분리 전략
- 서비스 간 통신 방식

**평가 기준**:
- [ ] 적절한 서비스 분리
- [ ] 데이터베이스 분리
- [ ] API 설계 고려
- [ ] 장애 격리 전략

### 5. 비즈니스 로직 연결도
**요구사항**:
- 주요 비즈니스 프로세스 플로우
- 사용자 여정 및 시스템 상호작용
- 데이터 플로우 및 의존성
- 외부 시스템 연동

**평가 기준**:
- [ ] 명확한 비즈니스 플로우
- [ ] 시스템 간 상호작용
- [ ] 데이터 흐름 표현
- [ ] 예외 처리 고려

### 6. 모니터링 다이어그램
**요구사항**:
- 로그, 메트릭, 트레이싱 구성
- 알림 및 대시보드 설계
- 장애 감지 및 대응 체계
- SLA/SLO 모니터링

**평가 기준**:
- [ ] 포괄적인 모니터링 커버리지
- [ ] 적절한 알림 설정
- [ ] 장애 대응 체계
- [ ] 성능 지표 정의

## 🎯 성공 팁

### 효율적인 팀워크
1. **명확한 역할 분담**: 각자의 강점에 맞는 다이어그램 할당
2. **중간 점검**: 20분마다 진행상황 공유
3. **일관성 유지**: 색상, 스타일, 명명 규칙 통일

### 다이어그램 품질 향상
1. **5분 설명 규칙**: 5분 안에 설명 가능한 수준으로 단순화
2. **스토리텔링**: 사용자 관점에서 시작해서 기술적 세부사항으로
3. **검증 가능성**: 실제 구현 가능한 현실적인 설계

### 발표 준비
1. **핵심 메시지**: 각 다이어그램당 1-2개 핵심 포인트
2. **연결성**: 6개 다이어그램 간의 연관성 설명
3. **차별화**: 다른 팀과의 차별점 강조
