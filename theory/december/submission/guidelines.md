# 제출 가이드라인

## 개요

이 문서는 2시간 다이어그램 워크샵의 실습 과제를 Discord에 제출하는 방법을 안내합니다. 모든 학생은 4개의 실습 과제를 완료하고 지정된 형식으로 제출해야 합니다.

## 제출 대상 과제

다음 4개의 실습 과제를 완료하여 제출하세요:

1. **실습 1**: AWS 3계층 아키텍처 (15분)
2. **실습 2**: Kubernetes 배포 (15분)
3. **실습 3**: CI/CD 파이프라인 (10분)
4. **실습 4**: 로직 플로우 (10분)

## 파일 형식

### 지원 형식

다음 두 가지 형식 중 하나를 선택하여 제출하세요:

#### PNG (권장)
- **장점**: 빠른 미리보기, 작은 파일 크기
- **해상도**: 최소 1920x1080
- **배경**: 투명 또는 흰색
- **품질**: 고품질 (100%)

#### PDF (선택사항)
- **장점**: 벡터 그래픽 유지, 인쇄 품질
- **페이지 크기**: A4 또는 Letter
- **방향**: 가로 (Landscape) 권장

### Draw.io에서 내보내기

#### PNG 내보내기
1. Draw.io에서 **File > Export as > PNG** 선택
2. 설정:
   - **Zoom**: 100%
   - **Border Width**: 10px
   - **Transparent Background**: 선택 (선택사항)
3. **Export** 클릭
4. 파일 저장

#### PDF 내보내기
1. Draw.io에서 **File > Export as > PDF** 선택
2. 설정:
   - **Page View**: Fit to Page
   - **Include a copy of my diagram**: 선택 (선택사항)
3. **Export** 클릭
4. 파일 저장

## 파일명 규칙

### 기본 형식

```
이름_과제번호.확장자
```

### 예시

- `홍길동_01.png` - AWS 3계층 아키텍처
- `홍길동_02.png` - Kubernetes 배포
- `홍길동_03.png` - CI/CD 파이프라인
- `홍길동_04.png` - 로직 플로우

### 규칙

1. **이름**: 본인의 실명 사용 (한글 또는 영문)
2. **과제번호**: 01, 02, 03, 04 (두 자리 숫자)
3. **확장자**: `.png` 또는 `.pdf` (소문자)
4. **구분자**: 언더스코어(`_`) 사용
5. **공백**: 파일명에 공백 사용 금지

### 잘못된 예시

❌ `홍길동 01.png` (공백 사용)
❌ `홍길동-01.png` (하이픈 사용)
❌ `홍길동_1.png` (한 자리 숫자)
❌ `홍길동_01.PNG` (대문자 확장자)
❌ `과제1.png` (이름 누락)

## 제출 방법

### Discord 채널

**제출 채널**: `#실습-제출`

### 제출 단계

1. **Discord 접속**
   - 워크샵 Discord 서버에 접속
   - `#실습-제출` 채널로 이동

2. **파일 업로드**
   - 메시지 입력창 왼쪽의 `+` 버튼 클릭
   - 또는 파일을 드래그 앤 드롭
   - 4개의 파일을 한 번에 업로드 가능

3. **메시지 작성** (선택사항)
   - 간단한 설명 추가
   - 특별히 고민한 부분 공유
   - 질문 사항 기재

4. **전송**
   - Enter 키 또는 전송 버튼 클릭

### 메시지 예시

```
[제출] 홍길동 - 실습 과제 1~4

AWS 아키텍처에서 Multi-AZ 구성을 표현하는 데 집중했습니다.
Kubernetes 다이어그램에서 ConfigMap과 Secret의 연결을 명확히 표시했습니다.

질문: CI/CD 파이프라인에서 병렬 실행을 표현하려면 어떻게 해야 하나요?
```

### 제출 확인

제출 후 다음을 확인하세요:

- [ ] 4개의 파일이 모두 업로드됨
- [ ] 파일명이 규칙에 맞음
- [ ] 이미지가 Discord에서 미리보기됨
- [ ] 파일 크기가 적절함 (각 파일 10MB 이하)

## 평가 기준

각 실습 과제는 다음 기준으로 평가됩니다:

### 1. 구조 (30점)

**평가 항목**:
- 다이어그램의 전체 구조가 논리적인가?
- 필수 구성 요소가 모두 포함되었는가?
- 계층 구조가 명확한가?

**점수 기준**:
- **25-30점**: 모든 필수 요소 포함, 구조 완벽
- **20-24점**: 대부분의 요소 포함, 구조 양호
- **15-19점**: 일부 요소 누락, 구조 개선 필요
- **0-14점**: 많은 요소 누락, 구조 불명확

### 2. 리소스 (30점)

**평가 항목**:
- 필수 리소스가 모두 포함되었는가?
- 리소스 간 관계가 정확한가?
- 리소스 속성이 올바르게 표시되었는가?

**점수 기준**:
- **25-30점**: 모든 리소스 정확, 관계 명확
- **20-24점**: 대부분의 리소스 정확, 관계 양호
- **15-19점**: 일부 리소스 누락 또는 부정확
- **0-14점**: 많은 리소스 누락 또는 부정확

### 3. 연결 및 흐름 (20점)

**평가 항목**:
- 리소스 간 연결이 논리적인가?
- 데이터 흐름 방향이 올바른가?
- 화살표와 레이블이 명확한가?

**점수 기준**:
- **17-20점**: 모든 연결 논리적, 흐름 명확
- **13-16점**: 대부분의 연결 논리적, 흐름 양호
- **9-12점**: 일부 연결 오류, 흐름 개선 필요
- **0-8점**: 많은 연결 오류, 흐름 불명확

### 4. 시각화 (20점)

**평가 항목**:
- 레이블이 명확하고 읽기 쉬운가?
- 색상 구분이 효과적인가?
- 정렬과 배치가 깔끔한가?
- 전체적인 가독성이 좋은가?

**점수 기준**:
- **17-20점**: 매우 명확하고 전문적
- **13-16점**: 명확하고 읽기 쉬움
- **9-12점**: 읽을 수 있으나 개선 필요
- **0-8점**: 읽기 어렵거나 혼란스러움

### 총점 계산

- **총점**: 100점 (각 과제 25점 × 4개)
- **합격 기준**: 60점 이상
- **우수**: 80점 이상

```
- 색상 사용이 효과적입니다

🔧 개선 필요:
- ALB와 EC2 간 연결에 보안 그룹 표시 추가

- CIDR 블록 레이블 추가

💡 제안:
- NAT Gateway를 추가하면 더 완전한 아키텍처가 됩니다
- Route Table을 표시하면 네트워크 흐름이 더 명확해집니다
```

## 자주 묻는 질문 (FAQ)

### Q1: 4개의 과제를 모두 제출해야 하나요?

**A**: 네, 4개의 과제를 모두 완료하여 제출해야 합니다. 부분 제출도 가능하지만, 완료하지 않은 과제는 0점 처리됩니다.

### Q2: 파일 크기 제한이 있나요?

**A**: Discord의 기본 업로드 제한은 파일당 10MB입니다. PNG 파일은 일반적으로 1-3MB 정도이므로 문제없습니다. 파일이 너무 크면 해상도를 조정하세요.

### Q3: 여러 번 제출할 수 있나요?

**A**: 마감 시간 전까지는 재제출이 가능합니다. 가장 마지막에 제출한 파일로 평가됩니다. 재제출 시 메시지에 "[재제출]"을 표시해주세요.

### Q4: 실습 시간 내에 완료하지 못했어요.

**A**: 워크샵 종료 후 30분의 추가 시간이 있습니다. 그 이후에는 다음 날 자정까지 제출 가능하지만 10% 감점됩니다.

### Q5: Draw.io 파일(.drawio)도 함께 제출해야 하나요?

**A**: 아니요, PNG 또는 PDF 이미지 파일만 제출하면 됩니다. 원본 .drawio 파일은 본인이 보관하세요.

### Q6: 영어로 작성해도 되나요?

**A**: 네, 한글 또는 영어 모두 가능합니다. 레이블과 설명이 명확하면 됩니다.

### Q7: 추가 도전 과제도 포함해야 하나요?

**A**: 아니요, 추가 도전 과제는 선택사항입니다. 기본 요구사항만 충족하면 됩니다. 추가 과제를 완료하면 가산점이 있을 수 있습니다.

### Q8: 다이어그램에 오류를 발견했어요. 어떻게 하나요?

**A**: 마감 시간 전이라면 수정 후 재제출하세요. 마감 후라면 `#질문` 채널에 문의하세요.

### Q9: 파일명을 잘못 지었어요.

**A**: 마감 시간 전이라면 올바른 파일명으로 재제출하세요. 마감 후라면 `#질문` 채널에 문의하여 수정 요청하세요.

### Q10: 평가 결과는 언제 확인할 수 있나요?

**A**: 제출 후 2-3일 이내에 Discord DM 또는 `#피드백` 채널을 통해 개별 피드백을 받을 수 있습니다.

## 기술적 문제 해결

### 파일 업로드 실패

**증상**: 파일이 업로드되지 않음

**해결 방법**:
1. 파일 크기 확인 (10MB 이하)
2. 파일 형식 확인 (PNG 또는 PDF)
3. 인터넷 연결 확인
4. Discord 앱 재시작
5. 웹 브라우저에서 시도

### 이미지 미리보기 안 됨

**증상**: 업로드는 되었으나 미리보기가 표시되지 않음

**해결 방법**:
1. 파일 형식 확인 (PNG 권장)
2. 파일 확장자 확인 (소문자)
3. 파일 손상 여부 확인
4. 다시 내보내기 후 재업로드

### 파일명 한글 깨짐

**증상**: 한글 파일명이 깨져서 표시됨

**해결 방법**:
1. 영문 파일명 사용 (예: `HongGilDong_01.png`)
2. 또는 Discord 웹 버전 사용
3. 메시지에 본인 이름 명시
