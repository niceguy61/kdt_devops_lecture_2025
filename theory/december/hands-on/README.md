# 실습 과제 가이드

## 개요

이 디렉토리는 2시간 다이어그램 워크샵의 실습 과제를 포함합니다. 각 과제는 Draw.io를 사용하여 실제 시나리오의 다이어그램을 작성하는 실습입니다.

**총 실습 시간**: 60분 (설정 10분 + 과제 50분)

## 학습 목표

이 실습을 통해 다음을 학습합니다:

1. **Draw.io 숙련도**: 전문적인 다이어그램 도구 사용법
2. **아키텍처 시각화**: 복잡한 시스템을 명확하게 표현
3. **커뮤니케이션**: 기술적 개념을 시각적으로 전달
4. **문서화**: 프로젝트 문서에 포함할 수 있는 다이어그램 작성

**파일**: [00_drawio_setup.md](00_drawio_setup.md)

Draw.io 기본 사용법과 AWS 라이브러리 설정을 학습합니다.

**학습 내용**:

- AWS 아이콘 라이브러리 추가

### 1. AWS 3계층 아키텍처 (15분)
**파일**: [01_aws_3tier.md](01_aws_3tier.md)

**주요 구성 요소**:
- VPC 및 서브넷 (퍼블릭/프라이빗)

- RDS (Multi-AZ)
- S3

**학습 포인트**:
- 고가용성 아키텍처 설계
- 네트워크 계층 분리

## 일반적인 팁

### 시간 관리
- 각 과제의 예상 시간을 지키세요
- 완벽보다는 완성을 목표로
- 시간이 남으면 추가 도전 과제 시도

### 다이어그램 작성
- **명확성**: 목적이 즉시 이해되도록
- **일관성**: 같은 요소는 같은 방식으로 표현
- **단순성**: 필요한 것만 포함

### Draw.io 활용
- **Ctrl+D**: 빠른 복제
- **Ctrl+G**: 그룹화로 관리
- **Arrange 메뉴**: 정렬 및 배치
- **스타일 복사**: 일관된 디자인

### 색상 사용
- **퍼블릭 리소스**: 초록색 계열
- **프라이빗 리소스**: 주황색 계열
- **네트워크**: 파란색 계열
- **에러/경고**: 빨간색 계열

## 제출 가이드

### 파일 형식
- **형식**: PNG (필수)
- **해상도**: 최소 1920x1080
- **배경**: 흰색 (투명 배경 사용 안 함)

### 파일명 규칙
```
이름_과제번호.png
```

**예시**:
- `홍길동_01.png` (AWS 3계층)
- `홍길동_02.png` (Kubernetes)
- `홍길동_03.png` (CI/CD)
- `홍길동_04.png` (로직 플로우)
