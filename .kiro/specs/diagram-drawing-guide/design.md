# 설계 문서: 2시간 다이어그램 워크샵

## 개요

이 설계 문서는 `theory/december` 디렉토리에 생성할 2시간 다이어그램 워크샵 자료의 구조와 내용을 설명합니다. 워크샵은 초급 DevOps 엔지니어가 Mermaid와 Draw.io를 사용하여 아키텍처 다이어그램을 작성하는 실용적인 기술을 습득하도록 설계되었습니다.

## 워크샵 일정

### 1부: 강의 (50분)
- 다이어그램 기초 (10분)
- Mermaid 소개 (10분)
- AWS 아키텍처 다이어그램 (10분)
- Kubernetes 다이어그램 (10분)
- CI/CD & 로직 플로우 (10분)

### 휴식 (10분)

### 2부: 실습 (60분)
- Draw.io 설정 및 기본 사용법 (10분)
- 실습 과제 1: AWS 3계층 아키텍처 (15분)
- 실습 과제 2: Kubernetes 배포 (15분)
- 실습 과제 3: CI/CD 파이프라인 (10분)
- 실습 과제 4: 로직 플로우 (10분)

## 정확성 속성

*속성은 시스템의 모든 유효한 실행에서 참이어야 하는 특성 또는 동작입니다. 속성은 사람이 읽을 수 있는 사양과 기계 검증 가능한 정확성 보장 사이의 다리 역할을 합니다.*

### 속성 반영

요구사항을 분석한 결과, 대부분의 요구사항은 문서 내용의 존재 여부를 확인하는 예제 기반 테스트입니다. 보편적인 속성은 다음과 같습니다:

**속성 1: Mermaid 코드-이미지 대응**
*모든* Mermaid 코드 블록에 대해, 해당하는 렌더링된 다이어그램 이미지가 존재해야 합니다
**검증: 요구사항 2.2**

**속성 2: 실습 과제 완전성**
*모든* 실습 과제에 대해, 요구사항, 단계별 가이드, 체크리스트가 포함되어야 합니다
**검증: 요구사항 8.2**

## 에러 처리

### 누락된 콘텐츠 감지

**전략**: 문서 빌드 시 콘텐츠 검증 체크

**에러 유형**:
1. **필수 섹션 누락**: 강의 자료 또는 실습 과제 파일 누락
2. **이미지 누락**: Mermaid 코드에 대응하는 렌더링 이미지 없음
3. **체크리스트 불완전**: 실습 과제에 체크리스트 누락

**처리 방법**:
- 검증 스크립트로 구조 확인
- 누락된 파일 목록 생성
- 빌드 실패 시 명확한 에러 메시지

### 시간 관리

**전략**: 각 섹션에 명확한 시간 할당

**에러 유형**:
1. **강의 시간 초과**: 50분 초과
2. **실습 시간 부족**: 60분 내 완료 불가

**처리 방법**:
- 각 섹션에 예상 시간 명시
- 선택적 내용 표시
- 우선순위 지정

## 테스팅 전략

### 단위 테스트

1. **문서 구조 테스트**
   - 모든 필수 파일 존재 확인
   - 파일명 규칙 검증
   - 디렉토리 구조 확인

2. **콘텐츠 검증 테스트**
   - 필수 섹션 존재 확인
   - 마크다운 문법 검증
   - 링크 유효성 확인

3. **이미지 검증 테스트**
   - 모든 참조 이미지 존재 확인
   - 이미지 파일 형식 확인
   - 이미지 크기 적절성 확인

### 속성 기반 테스트

**속성 1: Mermaid 코드-이미지 대응**
- **생성기**: 모든 마크다운 파일에서 Mermaid 블록 추출
- **속성**: 각 Mermaid 블록에 대응하는 이미지 파일 존재
- **검증**: 이미지 파일 존재 및 마크다운에서 참조 확인
- **반복**: 모든 마크다운 파일에 대해 실행

**속성 2: 실습 과제 완전성**
- **생성기**: hands-on 디렉토리의 모든 과제 파일 추출
- **속성**: 각 과제에 요구사항, 가이드, 체크리스트 섹션 존재
- **검증**: 필수 섹션 헤더 존재 확인
- **반복**: 모든 실습 과제 파일에 대해 실행

### 통합 테스트

1. **워크샵 플로우 테스트**
   - 강의 자료 순서 확인

   - 참조 링크 유효성 확인

2. **시간 할당 테스트**
